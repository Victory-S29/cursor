Основні функції git

Перелік основних функцій

  • Команди для налаштування git, що вказують ім'я та пошту користувача:
  • git config --global user.name _
  • git config --global user.email _
  • git init - створює пустий ґіт для проєкта
  • git status - описує статус проєкта - внесені/підтверджені зміни, файли, які під'єднані до git-а
  • git add - додає файл в робочий простір git
  • git commit - створює точку збереження робочого проєкта
  • git log - дозволяє отримувати інформацію про всі комміти проєкта
  • git checkout - функція для переходк між гілками проєкта
  • git branch - створює основну гілку проєкта
  • git push - відправляє проєкт на ґітхаб
  • git pull - функція для завантаження файлів репозиторія та автоматичного їх оновлення
  • git merge- об'єднує декілька гілок в одну
  • git clone- створює копію репозиторія

.gitignore - файл в каталозі репозиторія, в якому вказуються файли з якими не працює git

  • git revert - безпечний спосіб скасування операцій
  • git reset - скасовує локальні зміни в репозиторії
  • git rebase - об'єднує послідовність коммітів в один новий
  • git cherry-pick - команда, що дозволяє вибирати довільні комміти та додавати їх до головної гілки
  • git stash -локально зберігає всі останні зміни в робочій області та скидає стан робочої області до попереднього стану
